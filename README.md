# منصة مراقبة الدوام — مدينة الملك سلمان الطبية

نظام متكامل لمتابعة حضور الموظفين والجولات اليومية عبر المتصفح.

## الملفات

| الملف | الوصف |
|-------|-------|
| `index.html` | الصفحة الرئيسية — نقطة الدخول |
| `admin.html` | لوحة المسؤول الكاملة |
| `employee.html` | بوابة إدخال بيانات الموظف |
| `style.css` | التصميم الكامل |
| `script.js` | المنطق البرمجي (LocalStorage) |

## التشغيل المحلي

افتح `index.html` مباشرةً في أي متصفح حديث. لا يحتاج الملف إلى خادم.

## الرفع على GitHub Pages

1. أنشئ مستودعاً جديداً على [github.com](https://github.com)
2. ارفع جميع الملفات إلى المستودع
3. افتح **Settings → Pages** واختر الفرع `main` والمجلد `/root`
4. انقر **Save** — سيظهر الرابط خلال دقيقة تقريباً

```
https://<username>.github.io/<repo-name>/
```

## بيانات الدخول الافتراضية

| الحساب | البيانات |
|--------|---------|
| المسؤول | كلمة المرور: `admin1234` |
| الموظف  | يُنشئه المسؤول من لوحة التحكم |

يمكن تغيير كلمة مرور المسؤول من **لوحة التحكم → الإعدادات**.

## ترتيب الاستخدام

1. **المسؤول** يدخل لوحة التحكم ويضيف:
   - أقسام لكل مستشفى (إدارة الأقسام)
   - كوداً خاصاً لكل موظف (أكواد الموظفين)
   - جداول الدوام (اختياري)
2. **الموظف** يفتح `employee.html` ويُدخل الكود الخاص به
3. يملأ الموظف نموذج بيانات الدوام ويحفظه
4. **المسؤول** يعتمد التقارير أو يرفضها من سجل التقارير

## البيانات (وضع حالي: LocalStorage)

⚠️ **النظام يعمل حالياً بدون قاعدة بيانات خارجية.** جميع البيانات (الموظفون، الأقسام، جداول الدوام، التقارير، كلمة مرور المسؤول) محفوظة في **LocalStorage** بالمتصفح — تماماً كالإصدار الأول من النظام.

هذا يعني:
- لا حاجة لأي حساب Supabase أو إعداد خارجي — يعمل النظام فوراً بمجرد فتح `index.html`
- **قيد مهم:** بيانات كل جهاز/متصفح منفصلة عن الآخر. فإن أدخل موظف تقريراً من جهازه، والمسؤول فتح `admin.html` من جهاز أو متصفح مختلف، **لن يظهر له التقرير**. لعمل النظام كما هو مصمم (موظف يرسل ← مسؤول يعتمد)، يجب استخدام نفس الجهاز والمتصفح للطرفين، أو ربط النظام بقاعدة بيانات مشتركة لاحقاً.

### العودة إلى Supabase لاحقاً

الملفان `config.js` و`supabase-schema.sql` موجودان في المستودع وجاهزان لإعادة التفعيل مستقبلاً عند توفر حساب Supabase. الخطوات (عند الرغبة):
1. أنشئ مشروعاً على [supabase.com](https://supabase.com) ونفّذ `supabase-schema.sql` في SQL Editor
2. ضع بيانات مشروعك في `config.js`
3. أعد ربط `script.js` بطبقة Supabase (نسخة سابقة محفوظة في تاريخ Git لهذا المستودع)

## الأقسام الافتراضية

قائمة الأقسام لكل مستشفى **ثابتة داخل `script.js`** (كائن `DB.DEFAULT_DEPARTMENTS`) ولا تعتمد على أي قاعدة بيانات:

| المستشفى | الأقسام |
|---|---|
| الطب النفسي | الطوارئ، العناية المركزة، التنويم، العيادات الخارجية، المختبر، الصيدلية |
| النساء والأطفال | الولادة، الحضانات، الأطفال، النساء، الطوارئ، العمليات |
| المدينة الرئيسي | الطوارئ، التنويم، العناية المركزة، الجراحة، الباطنية، المختبر، الأشعة |

- **صفحة الموظف** تقرأ هذه القائمة مباشرة عند اختيار المستشفى — بدون أي استعلام.
- **لوحة المسؤول** (إدارة الأقسام) تُزرع بنفس القائمة تلقائياً أول مرة في LocalStorage، مع إمكانية إضافة أو حذف أقسام إضافية يدوياً من هناك.
