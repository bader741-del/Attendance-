# منصة مراقبة الدوام — مدينة الملك سلمان الطبية

نظام متكامل لمتابعة حضور الموظفين والجولات اليومية عبر المتصفح.

## الملفات

| الملف | الوصف |
|-------|-------|
| `index.html` | الصفحة الرئيسية — نقطة الدخول |
| `admin.html` | لوحة المسؤول الكاملة |
| `employee.html` | بوابة إدخال بيانات الموظف |
| `style.css` | التصميم الكامل |
| `script.js` | المنطق البرمجي (LocalStorage) |

## التشغيل المحلي

افتح `index.html` مباشرةً في أي متصفح حديث. لا يحتاج الملف إلى خادم.

## الرفع على GitHub Pages

1. أنشئ مستودعاً جديداً على [github.com](https://github.com)
2. ارفع جميع الملفات إلى المستودع
3. افتح **Settings → Pages** واختر الفرع `main` والمجلد `/root`
4. انقر **Save** — سيظهر الرابط خلال دقيقة تقريباً

```
https://<username>.github.io/<repo-name>/
```

## بيانات الدخول الافتراضية

| الحساب | البيانات |
|--------|---------|
| المسؤول | كلمة المرور: `admin1234` |
| الموظف  | يُنشئه المسؤول من لوحة التحكم |

يمكن تغيير كلمة مرور المسؤول من **لوحة التحكم → الإعدادات**.

## ترتيب الاستخدام

1. **المسؤول** يدخل لوحة التحكم ويضيف:
   - أقسام لكل مستشفى (إدارة الأقسام)
   - كوداً خاصاً لكل موظف (أكواد الموظفين)
   - جداول الدوام (اختياري)
2. **الموظف** يفتح `employee.html` ويُدخل الكود الخاص به
3. يملأ الموظف نموذج بيانات الدوام ويحفظه
4. **المسؤول** يعتمد التقارير أو يرفضها من سجل التقارير

## البيانات وقاعدة البيانات (Supabase)

جميع البيانات محفوظة الآن في **Supabase** (قاعدة بيانات PostgreSQL سحابية) بدلاً من LocalStorage، بحيث يشارك جميع الموظفين والمسؤولين نفس البيانات مهما اختلف الجهاز أو المتصفح.

### خطوات الإعداد (مرة واحدة فقط)

1. أنشئ حساباً ومشروعاً جديداً مجاناً على [supabase.com](https://supabase.com)
2. من داخل المشروع، افتح **SQL Editor** والصق محتوى ملف `supabase-schema.sql` بالكامل، ثم نفّذه (Run)
3. اذهب إلى **Settings → API** وانسخ:
   - **Project URL**
   - **anon public key**
4. افتح ملف `config.js` وضع القيمتين مكان `YOUR-PROJECT-REF` و`YOUR-ANON-PUBLIC-KEY`
5. ارفع جميع الملفات (بما فيها `config.js`) إلى GitHub Pages كالمعتاد

### ملاحظة أمنية

النظام لا يستخدم نظام مصادقة مستخدمين حقيقياً (Supabase Auth) — الحماية الوحيدة هي كلمة مرور واحدة للمسؤول، تماماً كما كان الحال سابقاً مع LocalStorage. هذا مناسب لاستخدام داخلي محدود، لكنه ليس بديلاً عن نظام صلاحيات حقيقي. راجع التعليقات في `supabase-schema.sql` لمزيد من التفاصيل.

## الأقسام الافتراضية

عند أول تشغيل للوحة المسؤول (وعندما تكون قائمة الأقسام فارغة)، يقوم النظام تلقائياً بزرع قائمة قياسية من الأقسام الشائعة لكل مستشفى من الثلاثة (راجع `DEFAULT_DEPARTMENTS` في `script.js`). يمكن للمسؤول بعد ذلك:
- حذف أي قسم من القائمة الافتراضية لا يحتاجه
- إضافة أي قسم إضافي يدوياً من **إدارة الأقسام → إضافة قسم**
