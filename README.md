# منصة مراقبة الدوام — مدينة الملك سلمان الطبية

نظام متكامل لمتابعة حضور الموظفين والجولات اليومية عبر المتصفح.

## الملفات

| الملف | الوصف |
|-------|-------|
| `index.html` | الصفحة الرئيسية — نقطة الدخول |
| `admin.html` | لوحة المسؤول الكاملة |
| `employee.html` | بوابة إدخال بيانات الموظف |
| `style.css` | التصميم الكامل (يشمل الوضع الليلي) |
| `script.js` | المنطق البرمجي (LocalStorage) |
| `manifest.json` | ملف تعريف تطبيق PWA |
| `sw.js` | Service Worker — عمل دون اتصال |
| `icon-192.png` / `icon-512.png` | أيقونات التطبيق |

## الميزات (النسخة 2.0)

- **توقيع النائب الإداري**: يُحفظ الاسم والتوقيع المرسوم من الإعدادات، ويُختم تلقائياً على كل تقرير معتمد ويظهر في النسخ المطبوعة
- **طباعة / PDF**: للتقارير اليومية والأسبوعية والشهرية مع خانة التوقيع في نهاية التقرير
- **نسخ احتياطي واستعادة**: تنزيل واستعادة جميع البيانات كملف JSON من الإعدادات
- **تنبيهات ذكية**: جرس التنبيهات يعرض التقارير المعلقة، جداول الدوام غير المغطاة اليوم، والنقص الحرج في التغطية
- **سجل تدقيق**: يوثّق كل إجراء (دخول، اعتماد، رفض، إضافة، حذف...) مع إمكانية التصدير
- **كلمة مرور مشفرة**: تُحفظ بتجزئة SHA-256 مع ترحيل تلقائي عند أول دخول
- **وضع ليلي**: زر تبديل في جميع الصفحات
- **PWA**: تثبيت المنصة كتطبيق على الجوال وتعمل دون اتصال (عند نشرها على HTTPS)

## التشغيل المحلي

افتح `index.html` مباشرةً في أي متصفح حديث. لا يحتاج الملف إلى خادم.

## الرفع على GitHub Pages

1. أنشئ مستودعاً جديداً على [github.com](https://github.com)
2. ارفع جميع الملفات إلى المستودع
3. افتح **Settings → Pages** واختر الفرع `main` والمجلد `/root`
4. انقر **Save** — سيظهر الرابط خلال دقيقة تقريباً

```
https://<username>.github.io/<repo-name>/
```

## بيانات الدخول الافتراضية

| الحساب | البيانات |
|--------|---------|
| المسؤول | كلمة المرور: `admin1234` |
| الموظف  | يُنشئه المسؤول من لوحة التحكم |

يمكن تغيير كلمة مرور المسؤول من **لوحة التحكم → الإعدادات**.

## ترتيب الاستخدام

1. **المسؤول** يدخل لوحة التحكم ويضيف:
   - أقسام لكل مستشفى (إدارة الأقسام)
   - كوداً خاصاً لكل موظف (أكواد الموظفين)
   - جداول الدوام (اختياري)
2. **الموظف** يفتح `employee.html` ويُدخل الكود الخاص به
3. يملأ الموظف نموذج بيانات الدوام ويحفظه
4. **المسؤول** يعتمد التقارير أو يرفضها من سجل التقارير

## البيانات والمزامنة السحابية (v2)

النظام **محلي-أولاً**: localStorage هو ذاكرة العمل، وSupabase (إن فُعّل) هو مصدر الحقيقة — سحب تلقائي كل دقيقة ودفع عبر صف انتظار يعيد المحاولة عند انقطاع الاتصال. ما دام `config.js` بقيمه النائبة يعمل كل شيء محلياً كما في v1.

### خطوات تفعيل السحابة

1. أنشئ مشروعاً على [supabase.com](https://supabase.com)
2. SQL Editor ← الصق `supabase-schema.sql` كاملاً ← Run
3. Authentication ← Add user ← بريد `admin@mksh.local` وكلمة مرور قوية
4. نفّذ سطر تسجيل المسؤول الموجود آخر ملف SQL (بعد تعديل البريد إن غيّرته)
5. Settings ← API ← انسخ Project URL و anon key إلى `config.js` وارفعه
6. ادخل كلوحة مسؤول — ستظهر رسالة رفع البيانات المحلية للسحابة لمرة واحدة

### الأمان

- RLS مفعّل على كل الجداول بسياسات أدوار (`is_admin()`) — لا سياسات مفتوحة
- المجهول يقرأ أسماء الأقسام فقط؛ التحقق من كود الموظف وإدخال التقارير عبر دوال RPC لا تكشف الجداول
- مصادقة المسؤول عبر Supabase Auth، وسجل التدقيق السحابي غير قابل للتعديل أو الحذف
